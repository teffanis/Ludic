🏗️ HTML - La Base de la Web

HTML (HyperText Markup Language) es el lenguaje fundamental para crear páginas web. Define la estructura y contenido utilizando elementos (etiquetas) que el navegador interpreta para mostrar texto, imágenes, formularios y más.

📝 Estructura

Define la esqueleto de la página con etiquetas como <header>, <main>, <footer>

🔗 Enlaces

Conecta páginas entre sí con la etiqueta <a>

📋 Formularios

Recibe datos del usuario con <form>, <input>, <textarea>

🎨 Compatibilidad

Funciona en todos los navegadores modernos sin necesidad de compilación

Ejemplo Básico 💻

HTML Básico: <!DOCTYPE html> <html> <head> <title>Mi Primera Página</title> </head> <body> <h1>¡Hola Mundo!</h1> <p>Este es mi primer párrafo en HTML</p> <a href="https://www.ejemplo.com">Enlace</a> </body> </html>

¿Cuándo usar HTML?

  • Para crear la estructura de cualquier sitio web
  • Presentar contenido (texto, imágenes, videos)
  • Crear formularios para recopilar información
  • Trabajar con SEO y metadatos

🎮 Demo Interactiva

Estructura básica de una página HTML:
  • <!DOCTYPE html> - Declara que es HTML5
  • <html> - Elemento raíz
  • <head> - Metadatos (título, estilos, scripts)
  • <body> - Contenido visible

☕ Java - Programación Orientada a Objetos

Java es un lenguaje de programación robusto, seguro y versátil. Se utiliza para crear aplicaciones empresariales, aplicaciones móviles (Android), sistemas backend y más. Su mantra: "Write Once, Run Anywhere" (WORA).

🔧 Multipropósito

Se ejecuta en cualquier máquina con Java Virtual Machine (JVM)

🏗️ POO

Orientado a objetos: clases, herencia, polimorfismo, encapsulación

🛡️ Seguridad

Tipado fuerte, manejo automático de memoria, control de acceso

⚡ Rendimiento

Compilación a bytecode, optimización en tiempo de ejecución

🖥️ Backend - El Corazón del Servidor

Backend es la lógica del servidor que procesa datos, maneja bases de datos, autentica usuarios y comunica con el frontend. Se utiliza con lenguajes como Node.js (JavaScript), Python, Java, PHP, Go, etc.

📡 APIs

Endpoints REST que comunican frontend y base de datos

🗄️ Bases de Datos

Almacenamiento seguro: SQL (PostgreSQL, MySQL) o NoSQL (MongoDB)

🔐 Seguridad

Autenticación, validación de datos, encriptación

⚙️ Escalabilidad

Manejo de múltiples usuarios y grandes volúmenes de datos

Ejemplo Básico con Node.js 💻

Node.js + Express - Servidor Simple: const express = require('express'); const app = express(); // Ruta GET app.get('/usuarios', (req, res) => { const usuarios = [ { id: 1, nombre: "Juan" }, { id: 2, nombre: "María" } ]; res.json(usuarios); }); // Ruta POST app.post('/usuarios', (req, res) => { const nuevoUsuario = req.body; // Guardar en BD... res.status(201).json(nuevoUsuario); }); app.listen(3000, () => { console.log('Servidor en puerto 3000'); });

¿Cuándo usar Backend?

  • Crear APIs para móviles o aplicaciones web
  • Procesar y almacenar datos del usuario
  • Implementar autenticación y autorización
  • Integrar con bases de datos
  • Lógica de negocio compleja

const express = require('express'); const app = express(); app.use(express.json()); let notas = [ { id: 1, titulo: "Nota 1", contenido: "Mi primera nota" } ]; // GET - Obtener todas las notas app.get('/notas', (req, res) => { res.json(notas); }); // POST - Crear una nueva nota app.post('/notas', (req, res) => { const nota = { id: notas.length + 1, titulo: req.body.titulo, contenido: req.body.contenido }; notas.push(nota); res.status(201).json(nota); }); // PUT - Actualizar una nota app.put('/notas/:id', (req, res) => { const nota = notas.find(n => n.id == req.params.id); if (nota) { nota.titulo = req.body.titulo || nota.titulo; nota.contenido = req.body.contenido || nota.contenido; res.json(nota); } else { res.status(404).json({ error: "No encontrada" }); } }); // DELETE - Eliminar una nota app.delete('/notas/:id', (req, res) => { notas = notas.filter(n => n.id != req.params.id); res.json({ mensaje: "Nota eliminada" }); }); app.listen(3000, () => console.log("Servidor en puerto 3000"));

Término Significado
SQL Structured Query Language - lenguaje estándar para consultar y manipular bases de datos relacionales
Base de Datos Conjunto organizado de datos almacenados y gestionados por un SGBD (Sistema Gestor de BD)
Tabla Estructura que organiza datos en filas (registros) y columnas (campos); similar a una hoja de cálculo
Columna (Campo) Atributo de la tabla con nombre y tipo de dato: VARCHAR, INT, DATE, BOOLEAN, DECIMAL, TEXT
Fila (Registro) Un conjunto de datos completo que representa una entidad; ej: los datos de un usuario específico
PRIMARY KEY Identificador único e irrepetible de cada fila en una tabla; generalmente un campo id autoincremental
FOREIGN KEY Campo que referencia la PRIMARY KEY de otra tabla para establecer relaciones entre tablas
UNIQUE Restricción que garantiza que ningún valor se repite en esa columna; ej: columna de email
NOT NULL Restricción que obliga a que el campo siempre tenga un valor; no puede quedar vacío
DEFAULT Valor que se asigna automáticamente si no se especifica al insertar: DEFAULT CURRENT_TIMESTAMP
AUTO_INCREMENT El valor del campo se incrementa automáticamente con cada nuevo registro (MySQL); SERIAL en PostgreSQL
SELECT Consulta datos: SELECT columnas FROM tabla WHERE condicion ORDER BY campo LIMIT n
INSERT INTO Inserta nuevos registros: INSERT INTO tabla (col1, col2) VALUES (val1, val2)
UPDATE Modifica registros existentes: UPDATE tabla SET columna=valor WHERE condicion
DELETE Elimina registros: DELETE FROM tabla WHERE condicion; sin WHERE elimina todo
CREATE TABLE Crea una nueva tabla con sus columnas, tipos y restricciones
ALTER TABLE Modifica la estructura de una tabla: añadir/eliminar columnas, cambiar tipos, agregar índices
DROP TABLE Elimina una tabla y todos sus datos permanentemente
WHERE Filtra registros según una condición: WHERE edad > 18 AND ciudad = 'Madrid'
ORDER BY Ordena resultados: ORDER BY nombre ASC (ascendente) o DESC (descendente)
LIMIT / OFFSET Limita la cantidad de resultados; OFFSET permite paginación: LIMIT 10 OFFSET 20
JOIN Combina filas de dos o más tablas relacionadas; tipos: INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL JOIN
INNER JOIN Retorna solo filas que tienen coincidencia en ambas tablas
LEFT JOIN Retorna todas las filas de la tabla izquierda y las coincidencias de la derecha (NULL si no hay)
GROUP BY Agrupa filas con el mismo valor para aplicar funciones de agregación como COUNT, SUM, AVG
HAVING Filtra grupos creados con GROUP BY; es como el WHERE pero para grupos: HAVING COUNT(*) > 5
Funciones de Agregación COUNT (contar), SUM (sumar), AVG (promedio), MAX (máximo), MIN (mínimo); operan sobre grupos de filas
LIKE Búsqueda de patrones en texto: WHERE nombre LIKE 'J%' (empieza con J), '%ez' (termina en ez)
IN / NOT IN Comprueba si un valor está en una lista: WHERE id IN (1, 2, 3)
BETWEEN Filtra valores en un rango: WHERE edad BETWEEN 18 AND 30
IS NULL / IS NOT NULL Comprueba si un campo es nulo; no se usa = NULL sino IS NULL
Subquery Consulta anidada dentro de otra: SELECT * FROM tabla WHERE id IN (SELECT id FROM otra)
Índice (INDEX) Estructura de datos que acelera las búsquedas en tablas grandes a costa de más espacio en disco
Transacción Conjunto de operaciones que se ejecutan como una unidad; o todas tienen éxito (COMMIT) o se revierten (ROLLBACK)
COMMIT / ROLLBACK COMMIT confirma los cambios permanentemente; ROLLBACK deshace todos los cambios de la transacción
Vista (VIEW) Consulta guardada que actúa como una tabla virtual; simplifica consultas complejas repetidas
Procedimiento Almacenado Bloque de código SQL guardado en la BD que puede ejecutarse con parámetros como una función
Trigger Código que se ejecuta automáticamente en respuesta a INSERT, UPDATE o DELETE en una tabla
Normalización Proceso de organizar tablas para eliminar redundancia e inconsistencia; formas normales 1FN, 2FN, 3FN
NoSQL Bases de datos no relacionales (MongoDB, Redis, Cassandra) para datos no estructurados o alta escalabilidad

🍃 MongoDB - Base de Datos NoSQL Orientada a Documentos

MongoDB es la base de datos NoSQL más popular del mundo. Almacena información en documentos JSON flexibles (BSON) en lugar de tablas rígidas, lo que la hace ideal para datos variables, aplicaciones modernas y escalabilidad horizontal. Es la "M" del stack MEAN/MERN.

📄 Documentos

Almacena datos como documentos JSON/BSON con estructura flexible y anidada

🚀 Escalabilidad

Sharding horizontal para distribuir datos en múltiples servidores

⚡ Alto Rendimiento

Lectura/escritura rápida, índices eficientes y agregaciones poderosas

🔄 Esquema Flexible

Sin esquema fijo; cada documento puede tener campos distintos
